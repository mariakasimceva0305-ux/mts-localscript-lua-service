# Сценарий демо-видео (2–3 минуты)

Цель: показать **одну команду запуска**, **локальность** (Docker + Ollama), основные эндпоинты и сценарии **generate / edit / clarification**, плюс упоминание **валидации и reflexion**.

---

## 0. Подготовка кадра (5–10 с)

- Окно терминала в **корне репозитория**.
- По желанию: второе окно — `docker compose ps` или Docker Desktop (видно контейнеры `ollama`, `lua-gen-api`).

---

## 1. Одна команда запуска (15–20 с)

**Команда:**

```bash
docker compose up --build
```

**Что говорить:**

- «Поднимаем **Ollama** и **API**; контейнер **ollama-pull** один раз скачивает **две** модели в volume — первый старт может быть долгим.»
- Дождаться, пока **`lua-gen-api`** станет **healthy** (или по логам `api` — строки готовности).

**Локальность:** всё крутится на машине; внешние LLM API не вызываются.

---

## 2. Health (5 с)

```bash
curl -sS http://127.0.0.1:8000/health
```

Ожидаемо: `{"ok":true}` — процесс API жив.

*Windows:* при необходимости `curl.exe` и `--noproxy "*"` (см. README).

---

## 3. Ready + модели (15–20 с)

```bash
curl -sS -w "\nHTTP %{http_code}\n" http://127.0.0.1:8000/ready
curl -sS http://127.0.0.1:8000/models
```

**Смысл:**

- `/ready` — Ollama доступен и в каталоге есть **хотя бы одна** из двух моделей (primary/fallback).
- `/models` — показывает имена моделей из env **без** вызова инференса.

---

## 4. Обычный generate (25–40 с)

```bash
curl -sS -X POST http://127.0.0.1:8000/generate \
  -H "Content-Type: application/json" \
  -d "{\"prompt\":\"верни значение wf.vars.userName\",\"context\":null}"
```

**Комментарий:**

- Ответ приходит с полем **`code`** и **`validation.syntax_ok: true`**.
- Упомянуть: перед этим внутри API отработали **retrieval** (чанки в системном промпте) и **`luac`**.

---

## 5. Edit mode (25–40 с)

```bash
curl -sS -X POST http://127.0.0.1:8000/generate \
  -H "Content-Type: application/json" \
  -d "{\"prompt\":\"Исправь: возвращай последний элемент списка, не первый\",\"context\":{\"previous_code\":\"return wf.vars.items[1]\"}}"
```

**Смысл:** в промпт попадает **предыдущий код** + формулировка правки; модель выдаёт обновлённый Lua.

---

## 6. Clarification (20–30 с)

```bash
curl -sS -X POST http://127.0.0.1:8000/generate \
  -H "Content-Type: application/json" \
  -d "{\"prompt\":\"сделай хорошо\",\"context\":null}"
```

**Ожидание:** `status: needs_clarification`, заполненное **`clarifying_question`**.

---

## 7. Валидация и reflexion (20–30 с, без углубления в код)

**Тезисы (без обязательного длинного live-примера):**

- После каждого успешного извлечения ```lua вызывается **`luac`** и доменные правила → `hard_errors` / `warnings` / `hints`.
- Если **`syntax_ok`** ложен после первого прохода — автоматически **второй** вызов модели (**reflexion**) с полным отчётом валидации; в ответе **`iterations: 2`**, **`reflexion_applied: true`**.
- На стабильных кейсах часто **`iterations: 1`** — значит первый проход уже прошёл проверку.

*(Опционально:* коротко показать логи контейнера `api` с тегами `[gen1]` / `[review]` / `[final]` после запроса, который триггерит reflexion — только если заранее подобран prompt.)*
