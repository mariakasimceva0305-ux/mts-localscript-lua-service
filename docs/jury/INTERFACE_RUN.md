# LocalScript Interface Run Guide (Freeze)

## Цель

Краткая инструкция по запуску и проверке именно интерфейса `LocalScript` в локальном контуре.

## Предусловия

- Запуск из **корня проекта**.
- Установлены Docker + Docker Compose.
- Для пересборки frontend vendor-bundle нужен Node.js + npm.

## Быстрый запуск (из корня проекта)

```bash
npm install
npm run build:vendor
docker compose up --build
```

После старта откройте в браузере:

- `http://127.0.0.1:8000`

## Команды (по шагам)

### 1) Сборка vendor для интерфейса

```bash
npm run build:vendor
```

Что делает команда:
- собирает `scripts/vendor-entry.js`;
- обновляет `api/static/vendor/app-vendor.js`;
- фиксирует используемые в рантайме редакторные и UI-зависимости без CDN.

### 2) Поднятие сервиса

```bash
docker compose up --build
```

Ожидание:
- API и связанные сервисы переходят в рабочее состояние;
- интерфейс доступен на `http://127.0.0.1:8000`.

## Что проверить после запуска

1. Страница открывается без пустого экрана.
2. В шапке видны бейджи статуса API/моделей и переключатели профиля/глубины.
3. Доступны режимы `Новый скрипт`, `Правка`, `Проверка`.
4. Редактор Lua отображается как центральный артефакт.
5. Нижняя панель переключается по вкладкам:
   - Проверка
   - Знания
   - Пояснение
   - Диагностика
   - Справка

## Ручные сценарии (обязательный минимум)

### Сценарий A: Базовая генерация

1. Нажать `Попробовать пример -> Простой скрипт`.
2. Нажать `Выполнить`.
3. Проверить:
   - код появился в редакторе;
   - обновился блок "Одним взглядом";
   - во вкладке "Проверка" есть корректный статус.

### Сценарий B: Уточнение

1. Нажать `Попробовать пример -> Уточняющий вопрос`.
2. Нажать `Выполнить`.
3. Проверить:
   - отображается блок `needs_clarification`;
   - видны подсказки и поле быстрого ответа;
   - кнопка продолжения отправляет уточнение.

### Сценарий C: Проверка кода

1. Загрузить/вставить код в редактор.
2. Нажать `Проверить редактор` (или включить режим `Проверка` и `Выполнить`).
3. Проверить:
   - списки ошибок/предупреждений/подсказок заполняются корректно;
   - статус "Проверка пройдена" показывается только при валидном результате.

### Сценарий D: Настройки

1. Открыть `Настройки`.
2. Изменить тему, масштаб UI и размер шрифта кода.
3. Сохранить.
4. Проверить:
   - изменения применились сразу;
   - после перезагрузки страницы значения сохранились.

## Как понять, что интерфейс работает корректно

Интерфейс считается корректным, если:
- не ломается главный путь "задача -> код -> проверка";
- режимы и статусы читаются без дополнительного объяснения;
- вкладки результата и блок уточнения стабильно отрабатывают;
- локальный профиль/гость и настройки работают логично;
- пустые состояния информативны и без пугающих ошибок.

## Минимальный smoke-check интерфейса (3-5 минут)

1. Открыть `http://127.0.0.1:8000`.
2. Нажать `Простой скрипт` -> `Выполнить`.
3. Переключить вкладки результата.
4. Запустить `Уточняющий вопрос` и отправить уточнение.
5. Открыть `Настройки`, сменить тему и шрифт кода, сохранить.
6. Переключить `Гость`/`Локальный профиль`, убедиться в понятном поведении.

Если все шаги прошли без сбоев — интерфейс готов к демо и защите.
